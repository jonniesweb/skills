# Session Retrospective

Reflect on this conversation from start to finish. Be honest, specific, and constructive. Address each section below.

## 1. What Worked Well

Identify specific moments, approaches, or tool uses in this session that were effective. Reference concrete examples from the conversation.

## 2. What Didn't Work

Identify missteps, wasted effort, wrong turns, or inefficiencies. Be specific about what happened and why it was suboptimal.

## 3. Tool & Rule Improvements

Based on this session, suggest specific improvements to:
- **Tools**: Are there missing capabilities, awkward workflows, or tools that didn't behave as expected?
- **Rules/Instructions**: Are there CLAUDE.md rules, skill definitions, hooks, or settings that should be added, changed, or removed to prevent issues seen in this session?
- **Automation**: Are there repetitive patterns from this session that could be automated?

## 4. Work Style Observations

Provide observations about the user's work style that affected the session:
- Were instructions clear or ambiguous?
- Were there decisions that could have been made earlier?
- Were there patterns in how tasks were scoped or communicated that helped or hindered progress?

## 5. Unanswered Questions

List important questions that came up (explicitly or implicitly) during this session that the user never answered. These are questions whose answers would have meaningfully changed the approach or outcome.
